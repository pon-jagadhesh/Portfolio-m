What is StrideAPI?

Stride API is a lightweight, annotation-based API documentation and testing framework for Java Servlets. It provides an automated way to generate structured API documentation, enabling developers to efficiently understand and test their API endpoints. The framework scans servlets, extracts metadata, and generates documentation with support for request parameters, headers, request bodies, and responses.

Configuration File (config.properties)

To configure the scanning mechanism, ensure that your config.properties file contains:

package.name=com.yourpackage.name
project.name=Your project Name
  • package.name: Defines the root package to scan for servlets.
  • project.name: Specifies the project name to be displayed in the documentation.
 NOTE:Place this config file inside java/main/src

Stride API scans all servlets annotated with @WebServlet and extracts relevant API metadata such as:

  • Endpoints: The URL patterns mapped to the servlets.
  • HTTP Methods: The supported HTTP request types such as GET, POST, PUT, DELETE.
  • Request Parameters: The expected query or form parameters.
  • Headers: The required and optional request headers.
  • Request Body Schema: The expected structure of JSON or other request payloads.
  • Response Structure: The status codes, response format, and sample responses.

Endpoint and Action Annotations

These annotations define API endpoints and describe the actions they perform (for Custom-HTTP methods).

@WebServlet

Specifies a servlet as an API endpoint.

@ApiAction

Describes a specific action performed by an endpoint.

@ApiAction(actionName="getUsers", description="Fetches all users from the database")
Example: 

@ApiAction(actionName = "GET")
protected void handleAction1(HttpServletRequest request, HttpServletResponse response) throws IOException {
   // your logic goes here 
}

Request Parameter Annotations

These annotations define the request parameters that an API method accepts.

@ApiParam

    Header Annotations

    These annotations specify required or optional HTTP headers for API requests.

    @ApiHeader

    Defines an expected header.

    @ApiHeader(name="Authorization", type="string")
    Example: 
    
  @ApiHeader(name = "Authorization", type = "string", required = true)
protected void handleAction1(HttpServletRequest request, HttpServletResponse response) throws IOException {
     // your logic goes here 
}

    Request Body Annotations

    Defines the expected structure for request payloads.

    @ApiBody

    Describes a request body schema.

    @ApiBody(name="UserRequest", type="JSON", schema="User")
    Example: 
    
@ApiBody(name = "UserRequest", type = "JSON", schema = UserRequest.class)
protected void handleAction1(HttpServletRequest request, HttpServletResponse response) throws IOException {
   // your logic goes here 
}

    Response Annotations

    Define the possible responses for an API endpoint.

    @ResponseParam

    Describes a single API response.

    @ResponseParam(responseCode="200", description="Success", schema=User.class)

    @ResponseParams

    Defines multiple response variations.

    @ResponseParams({
    @ResponseParam(responseCode="200", description="Success", schema=User.class),
    @ResponseParam(responseCode="400", description="Invalid request data")
})
    Example: 
    
@ResponseParams({
@ResponseParam(responseCode="200", description="Success", schema=User.class),
@ResponseParam(responseCode="400", description="Invalid request data")
})

    Example Servlet Implementation

    Here's an example of how to implement a servlet with Stride API annotations:

    package hello;

import java.io.BufferedReader;
import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

import utils.ApiBody;
import utils.ApiHeader;
import utils.ResponseParam;

@WebServlet("/user")
public class UserServlet extends HttpServlet {

    @ApiHeader(name="Authorization", type="string", required=true)
    @ApiParam(name="userId", type="int")
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {

        // Read request parameters
        int userId = Integer.parseInt(request.getParameter("userId"));

        // Fetch user data from a database or API
        User user = fetchUser(userId);

        // Check if user exists
        if (user == null) {
            response.sendError(HttpServletResponse.SC_NOT_FOUND, "User not found");
            return;
        }

        // Set response content type and write JSON response
        response.setContentType("application/json");
        response.getWriter().write(user.toJson());
    }
  
    @ApiParam(name="userId", type="int")
    @ApiParam(name="status", type="string")
    @ResponseParam(
      responseCode = "200",
      description = "Successful response",
      mediaType = "application/json",
      headers = {"Content-Type: application/json"},
      examples = {"{\"message\": \"Success\"}"}
)
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {

        // Read request parameters
        int userId = Integer.parseInt(request.getParameter("userId"));
        String status = request.getParameter("status");

        // Update user status in a database or API
        updateUserStatus(userId, status);

        // Set response content type and write JSON response
        response.setContentType("application/json");
        response.getWriter().write("{\"message\": \"Status updated\"}");
    }
}

    This servlet defines:

    • A GET /users endpoint.
    • A userId request parameter.
    • An Authorization header.
    • A request body (UserRequest).
    • Possible responses (200 OK, 404 Not Found).

    Scanning and Extracting API Information

    The ServletScanner component of Stride API is responsible for scanning all servlets and extracting metadata based on annotations. It captures details such as:

    • Servlet Names and URL Mappings
    • Supported HTTP Methods
    • Request Parameters and Headers
    • Request Body Structure
    • Response Details and Examples
